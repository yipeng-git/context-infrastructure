# Phase 1 Positioning Scorecard

目标：第 3 周末根据反馈选择 1 个主线定位和 1 个备选定位。这个表避免只凭感觉决策，也避免被单条高曝光内容误导。

## 候选定位

### A. AI-native 工作流与个人生产系统

一句话定位：

> 帮高频 AI 用户把零散工具使用升级成可复用、可验证、可商业化的工作系统。

适合载体：

- 个人账号：观点、案例、工作流分析。
- 开源项目：检查表、模板、CLI、规则库。
- 服务 / 产品：工作流诊断、系统搭建、团队培训。

潜在受众：

- AI-heavy 工程师、独立开发者、内容创作者、知识工作者。
- 已经使用 AI，但结果不稳定的人。
- 想把个人工作方式产品化的人。

### B. 普通人的 AI 副业与最小 offer

一句话定位：

> 帮普通人从“会用 AI”走到“找到真实需求并测试最小付费 offer”。

适合载体：

- 个人账号：更破圈、更容易获得泛流量。
- 模板工具：副业方向画布、offer 测试表、用户访谈问题库。
- 服务 / 产品：副业方向诊断、小班工作坊。

潜在受众：

- 想用 AI 做副业但方向混乱的人。
- 有技能但缺少商业判断的人。
- 想从内容、服务或小产品开始的人。

### C. 个人知识系统商业化

一句话定位：

> 帮知识工作者把个人知识系统从收藏整理升级成内容、产品和服务的生产线。

适合载体：

- 个人账号：知识系统、内容飞轮、认知资产。
- 开源项目：知识到内容到 offer 的看板模板。
- 服务 / 产品：知识系统诊断、内容系统搭建。

潜在受众：

- 有大量笔记和素材，但输出不稳定的人。
- 想打造个人 IP 的知识工作者。
- 想把经验沉淀为课程、咨询或产品的人。

## 评分维度

每项 1-5 分。第 3 周末对三个候选定位分别打分。

| 维度 | A | B | C | 评分说明 |
|---|---:|---:|---:|---|
| 写作自然度 |  |  |  | 你是否能持续写出具体、有判断的内容 |
| 受众反馈 |  |  |  | 评论、收藏、私信、转发、复述质量 |
| 技术差异化 |  |  |  | 是否能利用你的 Agentic AI / 工具构建优势 |
| 开源可行性 |  |  |  | 是否容易做成模板、脚本或小工具 |
| 付费信号 |  |  |  | 是否有人愿意约聊、问价格、描述具体痛点 |
| 长期兴趣 |  |  |  | 是否值得你连续做 6-12 个月 |
| 市场清晰度 |  |  |  | 目标人群和痛点是否足够明确 |
| 竞争区分度 |  |  |  | 是否能避开泛泛 AI 工具教学 |
| 总分 |  |  |  | 最高 40 分 |

## 决策规则

### 选择主线

满足以下条件之一即可成为主线：

- 总分最高，且付费信号不低于 3 分。
- 总分不是最高，但技术差异化 + 长期兴趣 + 付费信号三项合计最高。
- 三周内产生了明确强信号：有人提出具体需求、愿意约聊，或愿意试用模板。

### 选择备选

备选定位要满足两个条件：

- 与主线有明显协同，而不是完全分散精力。
- 能作为内容支柱或产品模块并入主线。

### 暂停规则

如果某个定位出现以下情况，第 4 周暂停投入：

- 只有点赞，没有评论、私信或复用行为。
- 写作需要大量外部输入，自己没有独特经验。
- 很容易落入泛泛教程、工具盘点或鸡血叙事。
- 很难提出一个具体 offer。

## 默认建议

如果前三周反馈没有明显差距，默认主线选择 A：AI-native 工作流与个人生产系统。

理由：

- 它最贴合你已有技术资产和真实经验。
- 它可以同时容纳账号、开源项目和服务。
- 它的变现路径更清楚：诊断、系统搭建、模板、工具、团队工作坊。

默认备选选择 C：个人知识系统商业化。

理由：

- 它可以作为 A 的一个内容支柱，而不是独立方向。
- 它能把你的 memory system、写作系统和内容资产沉淀连接起来。
- 它比 B 更容易形成差异化。

B 适合用来做破圈内容，但不建议一开始作为主线。这个方向竞争更泛，容易滑向“AI 副业教程”，需要更强案例和付费结果支撑。
