# Business / IP / Monetization Execution Hub

这是 2-4 个月商业/IP/变现计划的执行工作台。原计划文件不在这里修改；这里承载实际执行材料。

## 文件顺序

1. `week1_topic_map.md`
   - 第一周 3 个探索选题。
   - 每个选题都有一句可争议判断、内容角度、项目角度和变现角度。

2. `phase1_publish_pack.md`
   - 前三周 6 条发布草稿。
   - 包含反馈记录表，用于回填曝光、互动、私信、对话和变现信号。

3. `templates/weekly_feedback_review.md`
   - 每周复盘模板。
   - 用于记录输入、判断、内容反馈、项目反馈、用户对话和 offer 信号。

4. `phase1_positioning_scorecard.md`
   - 第 3 周末定位评分表。
   - 用来选择主线定位和备选定位。

5. `phase2_content_system.md`
   - 第 4-8 周内容系统。
   - 包含 3 个内容支柱、选题库、可复用素材、项目转化和 offer 转化。

6. `offers/phase3_offer_tests.md`
   - 第 9-12 周 offer 测试。
   - 包含 3 个轻量 offer、访谈脚本、5 次用户对话记录表和决策规则。

7. `phase4_scale_decision.md`
   - 第 13-16 周放大决策。
   - 根据内容、项目和付费信号选择服务化、开源项目、扩大分发或产品化。

## 执行原则

每周必须产生外部反馈。只学习、只写私密笔记、只打磨产品，都不算完成一周。

每个方向都同时看三类信号：

- 内容信号：观点是否能吸引对的人。
- 项目信号：模板、脚本、开源项目是否有人复用。
- 变现信号：是否有人愿意约聊、提供材料、问价格或付费。

前三周允许宽，第 4 周必须收敛。默认主线是 AI-native 工作流与个人生产系统，除非前三周数据明确支持其他方向。

## 本周最小动作

1. 从 `phase1_publish_pack.md` 选择 Draft 1 发布。
2. 发布后 24 小时和 72 小时各回填一次反馈表。
3. 同时准备 Draft 2，并把 `week1_topic_map.md` 里的 AI workflow audit template 想法整理成一个最小 README。
4. 周末复制 `templates/weekly_feedback_review.md` 做第一次复盘。
