# Week 1 Topic Map

目标：第一周不急着选择“个人账号 / 开源项目 / 产品”中的任何一种，而是用三个选题同时测试表达、项目和变现线索。每个选题都必须带一个可争议判断，因为只有可争议判断才会带来真实反馈。

## 选题 1：AI-native 工作者真正的商业机会在哪里

核心判断：

> AI-native 工作者的机会不在于“更会用工具”，而在于把自己的工作流变成别人可购买、可复用、可验证的系统。

为什么值得测：

- 这个判断贴合你已有的 Agentic AI、Context Infrastructure、工作流工具实践。
- 它天然连接三种载体：可以写成账号内容，可以做开源模板，也可以转成诊断服务。
- 它会筛出一批高价值读者：已经在用 AI，但感觉结果不稳定、缺少系统方法的人。

内容角度：

- 账号内容：为什么会用 Cursor / Claude Code 仍然不等于 AI-native 工作者。
- 开源项目：发布一个 `ai_workflow_audit_template`，帮人检查自己的 AI 工作流是否有输入、执行、验证、复盘闭环。
- 产品 / 服务：AI 工作流诊断，交付一份问题地图和改造建议。

要观察的反馈：

- 是否有人问“我的工作流能不能帮我看一下”。
- 是否有人收藏或转发给同事。
- 是否有人追问模板、脚本或工具。

## 选题 2：普通人用 AI 做副业，最大的瓶颈从执行转移到了哪里

核心判断：

> AI 把执行门槛压低之后，普通人副业失败的主要原因不再是不会做，而是不知道该为谁解决什么问题。

为什么值得测：

- 这个话题更破圈，适合测试个人账号的受众宽度。
- 它能把商业本质拉进来：需求、定位、渠道、付费意愿，而不是工具技巧。
- 它可以自然过渡到产品和服务：帮人从能力盘点走到最小 offer。

内容角度：

- 账号内容：AI 副业里最容易误判的一件事：以为会做内容 / 写代码 / 做图就有商业价值。
- 开源项目：副业 offer 画布模板，帮助用户把能力、目标人群、痛点、交付物、价格写清楚。
- 产品 / 服务：副业方向诊断，帮助用户从 3 个候选方向里选一个最小可测方向。

要观察的反馈：

- 评论区是否出现“那我应该怎么选方向”。
- 是否有人愿意填模板。
- 是否有人愿意约一次 30 分钟讨论。

## 选题 3：个人知识系统什么时候能从效率工具变成商业资产

核心判断：

> 个人知识系统只有在能持续生产外部可验证的产出时，才从效率工具变成商业资产。

为什么值得测：

- 这和你已有的记忆系统、context infrastructure、写作系统高度匹配。
- 它能连接个人 IP 和产品：内容系统、知识库、素材库、Agent 工作台。
- 它有明确反差：很多人把知识管理当收藏和整理，但商业价值来自输出和反馈。

内容角度：

- 账号内容：知识管理最大的问题不是信息太多，而是没有出口。
- 开源项目：个人知识商业化看板，追踪输入、判断、内容、反馈、offer 信号。
- 产品 / 服务：帮知识工作者搭建从素材到内容到 offer 的闭环。

要观察的反馈：

- 是否有人问模板和系统搭建。
- 是否有人分享自己的知识管理痛点。
- 是否有人愿意让你看他的现有系统。

## 第一周执行顺序

1. 先发选题 2。它更破圈，能快速测试普通受众对 AI 商业化话题的兴趣。
2. 再发选题 1。它更精准，能测试你的技术信誉和服务线索。
3. 最后发选题 3。它用于连接你的长期优势，并观察是否存在系统搭建需求。

## 第一周判定标准

第一周结束时，不按阅读量单独决策，而按四类信号综合判断：

- 表达信号：哪个选题让你写起来最自然，且观点最具体。
- 受众信号：哪个选题带来更高质量评论、私信或转发。
- 项目信号：哪个选题最容易转成小开源模板或工具。
- 变现信号：哪个选题最容易引发“能不能帮我看看”的请求。
