# 136. 全球大模型季报第9集：和广密聊，Coding是AGI第二幕、硅谷御三家真相、模型正成为新一代OS - Insights

## One-line Thesis

这期的核心判断是：`Coding Agent` 不是一个垂直产品机会，而是把模型从信息服务推向任务执行的 AGI 加速器。模型公司的竞争也因此从 C 端流量、benchmark 和单点范式，转向能否持续组织数据、算力、产品和高价值 token usage。

## Methodologies

- **用任务价值重排模型优先级**：评估 AI 机会时，不只看用户数和 DAU，而要看模型能否完成高价值任务、带来高频 token 消耗，并反哺训练数据。

- **把 Coding 当作 AI lab 的内部生产资料**：领先 Coding 模型会加速模型公司的研究、工程和数据迭代。一个模型公司如果依赖竞争对手提供 Coding API，它在研发生产力上相当于把关键 GPU 租给对手。

- **用 feedback loop 判断 AI 赛道成熟度**：Coding 先跑出来，因为代码有运行环境、测试、错误反馈和可验证结果。其他领域如果没有类似闭环，短期很难复制同等速度。

- **用组织能力判断模型公司，而不是只看 benchmark**：持续做好 SOTA model 需要每年数百亿美元投入、上百名顶尖研究者、管理层认知、数据工程耐心和产品转化能力。单次模型领先只是窗口，不是长期胜负。

- **用 token ROI 判断个人 AI workflow**：如果消耗 100 美元 token 能赚回 110 美元，说明 AI 使用开始进入商业闭环。只获得效率幻觉但没有收入、资产或可复用能力，闭环还没有成立。

## Thinking Patterns

- **从入口思维切到生产力思维**：ChatGPT 的 C 端入口价值很大，但 Coding Agent 直接改变生产函数。判断 AI 公司时，入口规模要和任务价值一起看。

- **从模型能力看组织文化**：Anthropic 的优势不只是模型强，而是能让聪明人长期做数据、环境、评估和产品细节。OpenAI 的优势也不只是模型强，而是 bottom-up 文化可能产生下一代范式。

- **从人类用户切到 Agent 用户**：未来工具的购买者和调用者可能是 Agent。产品设计要考虑 Agent 需要什么环境、权限、上下文和接口，而不只是人类如何点击页面。

- **从就业岗位切到成长链条**：AI 自动化 Junior 工作的风险，不只是减少岗位数量，更是截断人才从低级任务成长到高级判断的路径。

- **从单个应用切到操作系统生态**：模型如果成为 global GDP OS，Agent 就是新 OS 上的应用。应用公司有机会，但窗口会被模型平台的纵向整合持续挤压。

## Business Insights

- **Coding 的收入质量高于普通 C 端订阅**：少量塔尖开发者可以贡献远高于大众订阅用户的收入，因为他们把模型用于真实生产任务，愿意为更大模型、更高质量和更稳定服务付费。

- **Anthropic 的商业化来自高价格带定位**：它没有把模型做成廉价大众消费品，而是服务塔尖用户。这让大模型、高价格和高毛利能够同时成立，但前提是高端 Coding 体验持续领先。

- **OpenAI 的风险是过去胜利塑造了错误默认值**：ChatGPT 的成功让它自然追逐 DAU、订阅和 C 端增长。Coding 时代的关键变量变成高价值任务、数据闭环和 agentic 能力，这要求组织重新排序。

- **Google 的长期优势来自资源体系，短期问题来自产品和战略迟滞**：它有算力、现金流、TPU、Android 和 Workspace，但如果 Coding 决策晚三四个月，放大效应会让短期差距快速扩大。

- **Cursor 类应用公司的窗口取决于模型溢出**：当最强模型通过 API 外溢，Cursor 能吃到红利；当模型公司把最强能力优先放进自有产品，应用公司的护城河必须来自上下文、工作流、分发或自训模型。

- **Harness 是 Agent 时代的组织管理层**：模型能力决定上限，Harness 决定任务下限和可执行性。普通模型和开源模型借助好的 Harness，也能进入高价值任务。

- **模型公司可能成为新一代科技投资主线**：如果少数模型成为 global GDP OS，它们的收入和估值上限会远高于传统 SaaS。嘉宾由此把 AGI portfolio 更集中地压到头部模型公司。

- **AI 应用机会没有消失，但评估标准改变了**：未来一人公司仍可能成立，关键不是能否做出 demo，而是能否跑通 token 成本到收入的正向闭环。

## Actionable Applications

- 对个人 AI-native 工作流来说，先选一个能产生明确产出的任务，把 `token 成本 -> 产出资产 -> 收入或节省成本` 量化，而不是只记录“效率提升”。

- 对产品和工具来说，开始设计 `to agent` 的接口：稳定 API、权限边界、可恢复状态、可读日志、可验证输出，比漂亮 UI 更关键。

- 对内容和个人 IP 来说，护城河从“我会写”转向“我有判断、taste、问题选择和可验证工作流”。AI 会压缩执行成本，但不会自动给出独特立场。

- 对职业发展来说，尽快从 Junior 执行任务迁移到任务定义、系统设计、验收标准、数据组织和 Agent 编排。被自动化最快的是没有判断权的重复性知识劳动。

- 对投资和行业观察来说，少看单次发布热度，多看三件事：公司是否持续交付 SOTA model，是否拥有高价值 token usage，是否能把模型能力变成产品和生态。

## Open Questions

- Coding Agent 的领先壁垒到底有多高？如果难度系数低，OpenAI 和 Google 追上后 Anthropic 的塔尖市场会快速被压缩；如果难度系数高，Anthropic 可能继续拉开差距。

- C 端个人助理和高价值任务最终如何融合？如果 C 端分发仍然关键，ChatGPT、Gemini 和豆包会拥有额外优势；如果塔尖 token usage 更关键，Anthropic 路线会更强。

- 自动化 AI Researcher 是否真的会在一两年内出现？这决定了模型公司估值、白领就业冲击和 AI for Science 时间表。

- 一人公司能否规模化跑通收入？demo 门槛下降已经发生，但 token ROI、分发、信任和交付责任仍需要逐个验证。
