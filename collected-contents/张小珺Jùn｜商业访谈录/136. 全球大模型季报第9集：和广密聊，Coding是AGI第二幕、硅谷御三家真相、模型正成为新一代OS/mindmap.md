# 136. 全球大模型季报第9集：和广密聊，Coding是AGI第二幕、硅谷御三家真相、模型正成为新一代OS

- 00:00 这一季的核心矛盾：AI 加速进入 Agent 第二幕
  - Coding 从应用场景变成 AGI 加速器
    - Chatbot 第一幕解决对话和检索，Coding Agent 第二幕开始直接完成任务
    - 领先 Coding 模型类似领先 GPU，决定 AI lab 自身研发速度
    - 研究员和工程师开始从亲自写代码，转向提出任务、审阅结果和组织迭代
  - 进步速度超过社会消化速度
    - 过去一个季度的体感进步可能超过 2025 年全年
    - 白领工作进入通缩和失业窗口，社会认知没有同步更新
    - 本期的情绪是技术乐观和社会压力同时存在

- 03:51 Opus 4.5 到 4.6：模型从聊天能力跨向高价值任务
  - 关键变化是 Agentic 能力上来
    - 模型从回答问题变成执行复杂任务，任务价值量随之上升
    - 硅谷一线团队中，人写代码比例从七八成降到极低，人更多做 review
    - Claude Code 和 Codex 在部分任务上接近 CTO 或首席架构师水平
  - Coding 正在加速 AI 研究本身
    - 一个想法到跑通代码，从两三周压缩到一两天或几天
    - AI research 的部分突破来自 Codex 和 Claude Code，而不只是人类工程师
    - 多模态团队的数据处理 pipeline 也被 Coding Agent 缩短到几天或一周
  - 商业指标从 DAU 转向 token usage
    - Anthropic 用更少的塔尖用户贡献了接近或超过 OpenAI 大量 C 端订阅用户的收入
    - 超级开发者一周可消耗几千美元 token，成为更高价值的需求来源
    - 嘉宾判断模型公司 ARR 可能快速逼近新一代 MAG7 级别

- 11:16 Coding 为什么成为主线：代码是数字世界方案的压缩表达
  - 核心命题是语言即世界，代码即方案
    - 自然语言描述世界，代码描述解决方案，二者都有被验证过的强泛化能力
    - 如果代码能表达大多数数字任务，知识工作的自动化边界会被大幅外推
    - 数学、世界模型等方向可能重要，但泛化范围尚未像语言和代码一样被验证
  - Coding 不只是产品，而是 AI lab 的基础生产力
    - 不重视 Coding 的模型公司可能掉出第一梯队
    - 只给内部用的 Coding model 很难领先，因为内部数据分布不够全
    - 依赖竞争对手的 Coding API 类似依赖竞争对手 GPU，随时可能被断供
  - Coding 是亚马逊卖书式的起点
    - 卖书让亚马逊打通仓储、物流、用户和供应链，Coding 让模型打通任务、环境、评估和反馈
    - Coding 的反馈闭环最短，所以最先跑出高质量数据和产品收入
    - 长期看 Coding 只是 AGI 自动化全球 GDP 的一部分，但短期承担加速作用

- 14:12 AGI 三幕路线图：Chatbot、Coding Agent、AI Researcher
  - 第一幕是 Chatbot
    - 模型能对话、搜索和解释，但商业价值仍受限于信息服务
    - ChatGPT 的 C 端成功证明了模型入口价值，但没有覆盖最高价值任务
  - 第二幕是 Coding Agent
    - Agent 直接完成任务，带来高价值自动化和更高 token 消耗
    - 嘉宾认为 Coding Agent 做好后，大多数知识工作已经接近可自动化
    - 这一幕也会反向加速第三幕，因为 AI 可以加速 AI 研发
  - 第三幕是自动化 AI Researcher
    - 自动化研究员会面向基础科学、材料、脑科学和药物等领域
    - OpenAI 更想直接做爱因斯坦式研究员，Anthropic 更务实地先自动化白领工作
    - 时间表被显著压缩，可能今年底或明年初有公司宣布实现 AGI

- 17:11 做好 Coding 的难点：组织、文化和数据
  - 战略选择比单点技术更重要
    - Coding 需要几百个聪明人做数据、评估和环境构建等脏活苦活
    - AI lab 里很多人想做 0 到 1 突破，不愿长期投入数据工程
    - 管理层必须决定资源投向 C 端流量还是高价值任务
  - 数据壁垒变复杂
    - Chatbot 主要依赖文本对话数据，Coding Agent 需要任务、运行环境、反馈和评估
    - 数据质量和 data efficiency 比盲目扩大模型参数更关键
    - Coding 的数据飞轮会同时影响模型、产品和研发效率
  - C 端窗口和塔尖用户可能最终融合
    - C 端助理仍可能成为分发入口
    - 塔尖用户的 token usage 更像高价值数据和高价值收入
    - 强者生产力会进一步集中，社会分工随之改变

- 22:34 Anthropic：专注、数据文化和塔尖用户定位的胜利
  - All in Coding 是摸索后形成的战略聚焦
    - Sonnet 3.5 给了 Coding 正反馈，随后公司集中押注 Coding
    - 放弃 C 端、多模态等分散方向，把资源压到一条高价值任务路线
    - 创始团队重视数据、工程和细节，形成 top-down 的一致目标
  - 组织文化更像工业化球队
    - 不神化某个范式或单个明星，而是把每个环节都做好
    - 招募偏 underdog，文化面试严，团队更 AGI mission 驱动
    - 内部稳定、防泄密严格，人才流失相对少
  - 产品承接模型能力的效率高
    - Claude Code 没做 IDE，而是用终端形态承接模型能力的指数增长
    - 产品团队由工程师和研究员主导，更理解模型能力边界
    - 高价格带定位允许更大模型、更高效果和更高毛利
  - 风险来自算力和竞争追赶
    - 需求爆发超过算力规划，ARR 增长的瓶颈可能是 GPU
    - OpenAI 和 Google 大概率能追上 Coding 和 Agentic 能力
    - 塔尖市场能否长期守住，取决于 Coding 难度系数和迭代速度

- 33:41 OpenAI：C 端胜利变成 Coding 时代的路径依赖
  - ChatGPT 成功让 OpenAI 专注 C 端
    - 9 亿级周活和大量付费用户让 DAU、订阅和增长成为主叙事
    - 过去忙于和 Gemini 争 C 端，低估了 Coding 和高价值任务
    - 多模态和 Sora 等支线在 GPU 有限时占用了战略资源
  - 组织文化有探索优势，也有聚焦代价
    - bottom-up 文化可能让一两个人做出新范式突破
    - VC 式资源分配容易什么都想要，缺少对脏活累活的长期奖励
    - 重视 0 到 1 突破，弱化 1 到 100 的数据、产品和运营积累
  - OpenAI 仍可能阶段性被低估
    - 人才密度、算力资源和研究文化依然强
    - 新模型可能是真正的 GPT-5 级提升，Coding 和 Agentic 能力有机会追上
    - Chat、Code、Agent 一体化平台方向可能是正确策略
  - 对比 Anthropic 的两条路线
    - OpenAI 想做爱因斯坦，Anthropic 先自动化白领工作
    - Anthropic 的务实路线带来收入和资源，反过来加速 AGI
    - 今年格局仍可能交替领先，旧壁垒在模型自我迭代中不一定有效

- 47:18 Google Gemini：资源最厚，但 Coding 战略反应慢
  - Gemini 3.0 被嘉宾认为阶段性高估
    - Benchmark 很强，但 C 端持续增长和产品体验没有匹配热度
    - Google 证明自己不是 AI loser，但没有转化出足够收入和产品势能
    - 工程师文化强，PM 和产品组织相对弱
  - 最大失误是低估 Coding
    - 为争 C 端和多模态，晚了三四个月才把 Coding 提到最高优先级
    - Coding 落后三个月，可能在研发放大效应下演变成更长期落后
    - 不宜跳过 Coding 直接做数学或世界模型，因为 Coding 已经成为基础能力
  - 长期仍是最稳追随者
    - 算力、现金流、TPU、Android、Workspace 等资源短板少
    - 第三代职业经理人仍能让大公司机器持续运转
    - 只要战略跟随 OpenAI 和 Anthropic，掉队概率较低

- 54:13 Meta 和 xAI：挑战者的两种样本
  - Meta TBD 成为硅谷四号种子
    - 高薪挖人带来人才密度，9 到 10 个月交出还不错的模型
    - 技术上七八成学 Google，二成学 OpenAI，未来有机会追上第一梯队
    - 产品战略仍不清晰，更适合做低门槛个人助理或结合 Manus 做 Agent 产品
  - Manus 代表 Harness 产品窗口
    - Manus 被视为 Harness 的鼻祖，独立团队把 Agent 运行环境做得较完整
    - 如果更快做大收入，估值可能继续上升
    - 模型公司开始讲 Harness 后，应用公司窗口会受到模型平台挤压
  - xAI 的问题是战略摇摆和团队流失
    - 仅靠几十万卡和大力 scaling 解决不了数据质量问题
    - 从预训练、多模态、音频、搜索到 Coding 的反复切换会扰乱团队
    - FSD 的短反馈工程文化不等于大模型长期数据工程文化

- 01:01:39 Harness Engineering：Agent 需要自己的组织环境
  - Agent 应被当作一等公民
    - 人类知识工作者有电脑、环境、权限和信用卡，Agent 也需要相同级别的工作条件
    - 未来可能存在一个给 Agent 使用的平行工作世界
    - Harness 的目标是让 Agent 在组织环境里稳定完成任务
  - Harness 是模型之外的第二能力
    - 好的公司管理能提高人的下限，Harness 能提高普通模型的任务下限
    - 非 SOTA 模型和开源模型借助 Harness 也能做高价值任务
    - Claude 需求溢出会让更多模型被纳入 Agent 工作流
  - 产品对象从人类转向 Agent
    - 传统问题是 to C 还是 to B，新的问题是 to human 还是 to agent
    - Agent 可能决定调用哪些工具，token usage、token value 和 margin 变成关键指标
    - 工具生态会围绕 Agent 的调用偏好重新排序

- 01:03:48 中国模型公司和模型 OS 论
  - 国内公司开始追 Anthropic 路线
    - 部分国内模型公司更倾向高价值任务和 Coding
    - 豆包在 C 端更强，其他公司因 C 端难打而转向 Anthropic 路线
    - Coding 和 Agentic 能力不能输，最终仍拼组织能力和资源
  - AI 进入工业化竞争
    - 单个人才、单个方法或单次模型领先都不足以决定胜负
    - 持续做好 SOTA model 需要资金、算力、团队、管理层认知和产品策略共同成立
    - 新模型公司仍有机会，但难度接近再造一个台积电
  - 模型成为新一代操作系统
    - 操作系统的定义是支持应用无限扩展，Agent 会成为模型 OS 上的应用生态
    - 工作助理、生活助理、科研助理最终可能收敛到 global GDP OS
    - 最领先的几个模型会成为比 Google 更重要的技术基础设施

- 01:07:05 社会影响：知识和智力被压缩成 token
  - 白领工作价值被重新定价
    - 人类通过读书和知识获得工作的路径被模型压缩
    - Junior 程序员和工作两三年的岗位首先被自动化
    - 人才培养链条被截断，年轻人失去从初级岗位成长的环境
  - AI 同时带来通缩和失业压力
    - ChatGPT 和 Claude Code 降低了买顾问、买 SaaS 和买外包服务的需求
    - 微软等大公司可能用更少人完成更多产出
    - 技术繁荣和中产失业可能同时发生，社会矛盾会被放大
  - 人类剩余优势转向创造力和 taste
    - 有想法的人获得更低实现门槛，一个人或两个人可以做出过去大团队才能做的事
    - 审美、问题选择和创造力仍能决定产出质量
    - 嘉宾给普通人的建议是积极拥抱 AI，收益更多流向会使用 AI 的人

- 01:12:51 投资判断：模型公司成为 AGI portfolio 主线
  - 最核心指标是能否持续做好 SOTA model
    - 嘉宾更相信三五家领先模型公司，每家可能走向 10 万亿美元级别
    - 如果模型自动化全球 GDP 的 35% 到 50%，模型基金就是最直接表达
    - 投资组合应更聚焦模型，其他方向作为次级配置
  - 头部 AI 产品 ARR 说明增长速度
    - Anthropic 和 OpenAI 可能已达到数百亿美元 ARR 量级
    - Cursor、Perplexity、ElevenLabs、Suno、Manus、Lovable、GenSpark 等增长很快
    - 医疗方向如 OpenEvidence 等也被视为发展较快的应用层机会
  - 模型之外的高潜力方向
    - 机器人和 AI for Science 可能是模型之后的新平台
    - 机器人 6 到 18 个月内可能出现架构、路线和数据 scaling 的质变
    - 中国团队在硬件和产品创新上有优势，硅谷也开始重视深圳硬件能力

- 01:19:33 One-person company 和个人行动指标
  - 模型平台越强，个体越可能放大产出
    - 从 idea 到代码跑通再到 revenue 的路径被压缩
    - 一人公司可能成为常态，而不是少数神话
    - 自媒体、创作者和小团队会像过去借助微信、抖音一样借助模型平台
  - 最朴素的闭环是 token ROI
    - 消耗 100 美元 token，能否赚回 110 美元，是判断 AI workflow 是否有效的指标
    - 只消耗 token 不形成收入或资产，说明闭环还没有跑通
    - 嘉宾自己的状态也从 ChatGPT 转向 Claude Code，但仍没有达到塔尖开发者的 token 消耗强度
