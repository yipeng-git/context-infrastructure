# 138. 对罗福莉3.5小时访谈：AI范式已然巨变！OpenClaw、Agent范式很吃后训练、卡的分配、组织平权 - Insights

## One-line Thesis

这期的主线是：Agent 时代的竞争对象已经从单个模型，变成模型、Agent 框架、context、RL Infra、芯片和组织环境的组合系统。OpenClaw 让这套系统第一次被一线模型团队以可修改、可共创、可训练的方式看见，MiMo-V2 则是在长上下文推理效率和后训练敏捷性上提前押注。

## Methodologies

- **把 Agent 框架当作模型能力的一部分**：评估模型时不能只看 benchmark，要看它在真实 Agent 框架里的任务完成率。框架会通过 memory、context、工具调度、message channel 和成本路由改变模型的稳定性。

- **用高强度体验驱动组织转向**：罗福莉让团队密集使用 OpenClaw，不是为了完成考核，而是让每个人亲自看到任务边界被改写。新范式转向时，体验比宣讲更能统一判断。

- **先用顶尖模型改框架，再把框架复用给便宜模型**：Claude Opus 4.6 负责探索上限和改写 Agent 架构，沉淀下来的 memory、workflow、skills 和工具链再让中层模型稳定运行。这样做把 expensive exploration 和 cheap execution 分开。

- **把 Skills 视为隐性知识的数据接口**：企业规范、个人经验和组织方法往往不在互联网预训练数据里。通过人和 Agent 多轮协作，把这些信息写成 Skills，可以显著提高复杂任务的执行成功率。

- **围绕真实长程任务做 post-train**：Coding 的价值在于长上下文、环境反馈、可验证结果和多轮修正同时存在。Agent post-train 应该在端到端任务中训练，而不是只在特定 benchmark 上优化字符串输出。

- **用 3:1:1 思考 Agent 时代算力配置**：她给出的合理比例是研究:预训练:后训练约 3:1:1。这里最关键的变化是 post-train 不再是预训练后的短尾工作，而是和 pre-train 同等重要的长期投入。

- **为不确定的后训练周期保留架构冗余**：模型结构如果过度贴合某一种芯片、推理方式或场景，可能在半年到一年的 post-train 周期中失去灵活性。更简洁、可适配、留有算力富裕度的结构，在 Agent 时代更有价值。

## Thinking Patterns

- **从模型本体转向系统闭环**：她反复强调模型和 Agent 框架互相适配，说明前沿竞争已经不是单独问“模型强不强”，而是问“这套系统能否在复杂环境里持续完成更高价值任务”。

- **从 DAU 转向 token usage 和高价值任务**：DAU 代表分发和流量，但无法直接说明模型能否替代高智力工作。token usage、长 context、复杂任务完成率更接近 Agent 时代的能力指标。

- **从经验优先转向环境优先**：她认为经验可以在一两个月内被环境刷新，因此更看重基础、好奇心、热爱和上限。这里的环境包括更高标准目标、同伴互相蒸馏、真实任务压力和可用的算力。

- **从单人智能转向群体智能**：OpenClaw 社区、团队群聊、多人改框架、多人沉淀 Skills，本质都在把人类分散的 context 输入到 Agent 系统里。AGI 的路径不是模型独自进化，而是先吸收全人类的群体智慧。

- **从明确 reward 转向价值观约束**：量化时代价格是 reward，大模型时代的 reward 更模糊。她把“让世界更好一点”“替代无聊工作”作为方向约束，这其实是前沿技术团队面对模糊目标时的内在对齐机制。

## Business Insights

- **模型公司的产品边界会向 Agent 框架扩张**：当框架本身决定任务完成率，模型公司自然会重新进入产品层。模型加 Agent 架构，会变成新的产品形态。

- **成本是生产力革命的必要条件**：高价值任务可以承受昂贵模型，但普遍替代工作必须满足明显的成本收益比。便宜模型、端云混合、本地推理和框架补短板，会共同决定普及速度。

- **推理芯片需求可能被 Agent 直接放大**：模型能力提高、Agent 框架增强、成本下降之后，推理 token 需求会快速上升。她判断几倍到十倍的增长可能在很短时间内发生。

- **开源是加速生态的策略，不只是市场动作**：如果未来算力、芯片和部署高度分散，开源模型和开源框架会帮助更多生态位参与 Agent 进程。公司敢不敢开源，取决于它是否拥有模型之外的战略资源。

- **创业公司的最小规模会下降**：Agent 成熟后，几个人甚至一个人可以管理多个 Agent 作为生产力单元。短期瓶颈仍是模型成本、multi-agent 架构、自迭代和任务验收。

- **组织设计本身变成 AI Lab 的核心能力**：Agent 时代后训练需要多样性、快速重组和跨领域协作。传统固定组别可能压制创造力，扁平组织和热爱驱动在前沿研究里更有效。

## Actionable Applications

- 个人或团队做 AI-native workflow 时，优先建设 Agent 框架的中间层：memory、skills、工具路由、任务验收、成本路由和复盘机制。Prompt 只是入口，框架才决定长期复利。

- 在自己的项目里区分 `exploration model` 和 `execution model`：用最强模型设计架构和沉淀规则，再用更便宜模型沿着框架执行可重复任务。

- 把每次高质量人机协作沉淀成 Skills，而不是停留在单次 chat。真正有价值的是让隐性经验可复用、可迁移、可被其他 Agent 调用。

- 评估 Agent 不要只看一次 demo，要设计真实长程任务：多文件、多上下文、多轮反馈、有明确验收标准、有失败恢复路径。

- 组织 AI 采用时，不要只发工具链接。更有效的是制造共同体验和共享场景，让大家看到别人怎么用，再把用法沉淀成团队规范。

- 做长期技术判断时，把模型能力、框架能力、推理成本、芯片供给和组织敏捷性放在同一个模型里看。Agent 时代的领先很可能来自这些变量的乘积。

## Open Questions

- OpenClaw 展示的是一个可持续的 Agent 框架范式，还是某个阶段借助 Claude Opus 4.6 形成的强体验窗口？

- Agent RL 的 reward、环境复现和失败归因如何标准化，才能从少数团队的工程直觉变成可复用基础设施？

- Multi-agent 最终能否提高能力上限，还是主要停留在节省时间、节省成本和并行探索？

- 多模态到底是 AGI 的必要路径，还是 Agent 框架可以通过模型编排绕开单一多模态大模型的必要性？

- 当 AI 可以训练 AI 后，人类研究者最有价值的位置在哪里：提出问题、定义价值、提供数据，还是承担最终验收？
