# 138. 对罗福莉3.5小时访谈：AI范式已然巨变！OpenClaw、Agent范式很吃后训练、卡的分配、组织平权

- 00:00 这一期的总命题：大模型战争从 Chat 进入 Agent 第二幕
  - 竞争重心从预训练转向后训练
    - Chat 时代主要激发短上下文对话能力，Agent 时代要在复杂环境里完成长程任务
    - Pre-train 的代差正在缩小，Post-train 尤其是 Agent RL scaling 成为新战场
    - 模型、Agent 框架、RL Infra、算力和组织方式开始合成一套系统
  - 罗福莉的核心判断
    - Anthropic 路径已经成为前沿团队的共识
    - 国内团队如果反应足够快，追上当代 Claude Opus 4.6 可能只差两三个月
    - 未来半年谁领先，取决于现在怎么判断更大规模 scaling 和芯片路线

- 02:16 OpenClaw 引发巨变：从产品体验到 Agent 框架范式
  - 初始认知来自使用体验的突然转变
    - 罗福莉最初把 OpenClaw 看成 Claude Code 加 IM UI，对其运营化包装有抵触
    - 实际使用后，先被情绪感知、时间感知、context 编排和持续提醒打动
    - 三天内从生活管理、团队筛选、研究任务一路试到 user agent 构造，认知发生重估
  - OpenClaw 的价值不只是壳
    - 它把 memory、context、message channel、模型调度和任务执行规范组织成厚中间层
    - 对普通用户可见的是 UI，真正改变任务完成率的是框架如何给模型补足行动 context
    - 它不是单纯释放顶尖模型能力，也让中层模型在大量生活和泛代码场景里变得更稳定
  - 开源可改性带来新的创造力
    - Claude Code 的框架是黑盒，OpenClaw 允许用户修改 memory、workflow 和 multi-agent 逻辑
    - 顶尖模型可以先把框架改好，再让更便宜的模型沿着改好的框架运行
    - 这让模型进步和 Agent 框架进步变成互相推动的过程

- 17:11 Agent 框架的本质：人和模型之间的厚中间层
  - 框架定义的不只是交互层
    - 它决定给模型送什么静态信息，例如 memory、规则和技能
    - 它决定任务过程中的动态信息流，例如工具调度、远程控制、成本路由和任务拆分
    - 它还决定模型在不同场景下如何知道自己的长板和短板
  - 好框架在弥补行动缺陷
    - 持久化记忆补足跨 session 的连续性
    - 定时任务、心跳任务和主动检查补足模型缺少现实行动能力的问题
    - 更丰富的 message channel 让日常、管理、研究和泛代码任务进入同一个执行系统
  - 自学习不只发生在模型参数里
    - 模型能力提升后，会反过来改写 Agent 架构、memory 和执行规范
    - 框架提供的 context 也会随着训练和使用过程调整
    - 真正的自迭代需要模型、框架和人的反馈同时被吸收

- 24:17 群体智能提升 Agent 框架：从个人惊艳到团队躁动
  - 团队采用的关键是先制造体验
    - 春节后她要求团队高强度使用 OpenClaw，本质是让每个人亲自感受新范式
    - 团队在群里共享任务和用法，个体想象力被其他人的实践激发
    - 几周内完成过去可能需要数十周才能完成的研究准备和训练探索
  - 群体改框架比个人改框架更快
    - 近百人同时使用和修改同一套开源框架，几小时就能推进一轮改进
    - 每个人的场景都成为框架测试集，框架在真实任务中被逼出边界
    - 这种共同改进一套 Agent 框架的过程，被她视为 AGI 到来前的必要前兆
  - Skill 是人类隐性知识进入 Agent 的接口
    - 企业规范、业务逻辑、组织经验通常不在预训练数据里
    - 人和 Agent 多轮协作后，可以把经验沉淀成执行规范和 Skills
    - OpenClaw 带火 Skills 社区，让更多另类信息变成模型可调用的任务成功条件

- 31:51 研究范式变化：Coding 是长程任务和可验证反馈的入口
  - 研究效率被 Agent 大幅压缩
    - 过去从 idea 到评估和代码实现可能需要一两周，现在可并行交给多个 sub-agent
    - 十个研究 idea 可以同时验证，瓶颈从写代码转向 token、GPU 和实验排队
    - 如果研究 taste 准，Agent 放大的是正确方向上的迭代速度
  - Coding 有强泛化性的原因
    - 代码数据具备长上下文、强文件关联和明确环境反馈
    - 软件开发天然是长程任务，训练它会带来长上下文、工具使用和多轮修正能力
    - 代码任务可验证，所以适合同时服务 SFT、RL 和 Agent 框架迭代
  - 从 Code 外延到更广泛生产力任务
    - Code 做上限，其他领域训练保下限
    - 关键难点在于还原真实用户环境，否则 reward 和长程交互都会失真
    - 复杂任务不一定需要填满 1M context，但需要模型能在长 context server 中保持稳定

- 41:31 2026 是生产力变革之年：高价值任务和普惠智能分化
  - OpenClaw 在中国更火的原因
    - 国内开发者对效率提升更敏感，愿意用复杂工具换生产力
    - 国内有大量便宜好用的中层模型，API 成本和替代生产力之间的收益比更明显
    - 10 元 API 完成 1000 元价值任务，会显著提高用户试用动力
  - 去年的 Agent 还没有工业级可用
    - 早期 Agent 主要是复杂 system prompt 加一点环境反馈
    - 真正可用的 Agent 要能接入 Claude Code、OpenClaw 等复杂框架并稳定完成真实任务
    - 新的人机交互不再是逐行改代码，而是澄清需求、参与架构、提供业务 context
  - 两条发展路线同时存在
    - 前沿模型路线追求替代更高价值任务，带来更长 context 和更高 token usage
    - 普惠路线追求让所有人感受到智能水平，多模态、低成本和本地推理会更重要
    - 生产力革命的底层约束是成本、速度和端到端完成率

- 54:39 Agent 的自进化与自迭代：模型、框架和人一起变化
  - 未来还缺两层自进化
    - Agent 框架自身如何根据任务反馈持续改写
    - Agent 和人之间如何形成更自然、更高频的共同进化
    - 目前更多是在做模型适配 Agent，和 Agent 适配模型之间的双向流动
  - AI 可能先吸收人类智能，再产生更强智能
    - 她原本认为模型训练足够创造性，不会被 skill 化或 workflow 化
    - 近期体验让她相信，模型只要拿到研究 context，就可能复原研究路径并参与训练更强模型
    - AI 训练 AI 被视为未来一两年可能发生的关键转变
  - Multi-agent 仍在早期
    - 现在 multi-agent 更确定的价值是节省时间和成本
    - 它是否能提升任务完成上限，罗福莉还没有看到足够证据
    - 多 Agent 的自迭代、互相沟通和协同 RL 仍是开放问题

- 01:19:39 MiMo-V2 的伏击：多模型生态被 Agent 框架重新解释
  - 三个模型的意义来自编排
    - Pro 面向复杂理解和调度
    - Omni 面向感知，覆盖视频、音频、图片和文本
    - TTS 面向声音输出，承担表达层能力
    - OpenClaw 让这些原本分散的模型方向突然可以被组织进同一个 Agent 系统
  - 为什么不把所有能力合成一个模型
    - 语音生成、视觉理解和复杂推理的成本、速度、参数规模要求不同
    - Agent 革命关注端到端完成率、成本效率和速度，不追求单模型形式统一
    - 多模型同生态训练的好处是背景知识一致，任务交接更容易
  - MiMo-V2 的底层押注是长上下文推理效率
    - Flash 和 Pro 的结构目标是让长上下文建模效果好、推理速度快、KV cache 成本低
    - Hybrid attention 加 MTP 让模型在长 context、缓存命中和吐 token 速度上更适合 Agent
    - 她认为架构目标不能过多，否则后训练周期拉长后，早期约束会变成错误约束

- 01:26:15 架构选择：Hybrid Attention、MTP 和长上下文效率
  - Hybrid attention 的优势
    - Sliding window 层减少 KV cache，Full attention 层保留全局信息整合能力
    - Pro 把 Full attention 与 Sliding window 的比例推到更极致，目标是在更大模型上保持长文效率
    - 更大模型可以承受更稀疏的 attention，但小模型太稀疏会明显掉效果
  - MTP 的作用
    - 预训练中加入 MTP 可以提升基础能力
    - 推理时如果预测命中率高，MTP 通过 verify 机制降低单 token 成本
    - MiMo 的结构留下了大量计算富裕度，MTP 正好把这部分算力利用起来
  - 她对 MLA 的判断
    - MLA 在 Chat 时代和部分长文场景里是优秀结构
    - 但当 Post-train 周期拉长、Agent 场景快速变化时，过度贴合某种推理卡和场景会降低后续适应性
    - 更简洁、留有冗余的结构，可能更适合不确定的 Agent 时代

- 01:45:24 1T 是 Agent 时代的入场券：算力瓶颈重新排序
  - 1T 的意义是接近 Claude Opus 4.6 水平
    - 她认为要做到顶尖 Agent 能力，总参至少需要 1T 以上
    - 更关键的是激活参数，但激活参数越大，推理成本越高
    - 1T 不是终点，只是进入下一轮竞争的门槛
  - 卡的分配逻辑发生变化
    - Chat 时代预训练和后训练算力比例可能是 3:1 或 5:1
    - Agent 时代顶尖团队的 pre-train 与 post-train 可能接近 1:1
    - 她认为合理的卡分配是研究:预训练:后训练约 3:1:1
  - 训练 1T 的主要挑战是发现和解决细问题
    - Loss spike、expert 负载异常、激活值异常、算子 bug 都可能导致训练不稳定
    - 团队需要监控模型参数内部发生了什么，而不是把 spike 当正常波动
    - 停几天甚至两周排查问题，在她看来比带着隐患继续训练更合理

- 01:52:33 组织平权：大模型团队像创业团队而不是传统大厂部门
  - 小团队比大团队更适合深度排查
    - MiMo 训练链路真正投入一代模型的人大约二三十人到三四十人
    - 发现潜在问题后深入排查，需要关键环节的人高效协作，而不是多层级传递
    - 大团队在并行探索上有优势，但在深度问题定位上可能变慢
  - 没有固定组别是为了保护创造力
    - 不固定分预训练组、后训练组、数据组，让人随阶段自由流动
    - 做预训练的人转向后训练，有助于把数据多样性的视角带入 Agent post-train
    - 项目可以有人推动，但推动者不拥有对参与者的绝对控制权
  - 管理方式是热爱驱动
    - 她更相信筛选基础好、好奇心强、使命感强的人
    - 通过体验新事物激发热情，而不是靠考核动作本身
    - 钱是 baseline，价值感和意义感是更长期的驱动

- 02:02:56 训练细节与成本：Post-train 时代的定价和评估变化
  - 定价从推理成本转向任务价值
    - 预训练时代按架构和推理成本定价相对合理
    - 后训练时代，模型理解 Agent context 的能力会创造额外价值
    - Pro 抛弃 Flash 的极致低价逻辑，是因为能力溢价来自 post-train 和任务完成率
  - 1M context 的训练难点在数据
    - 理论上只要有足够多真实 1M 长上下文数据，模型可以学上去
    - 现实难点是构造稠密监督信号的成本太高
    - 长上下文能力是在预训练准备和后训练激活中缓慢提升的
  - Agent RL Infra 与 Chat Reasoning 不同
    - Chat reasoning 的 RL infra 以 rollout 推理引擎为核心
    - Agent RL infra 要以 Agent 为核心，处理框架、环境、CPU/GPU、存储、超时和异构资源
    - 训练和推理的不一致性在 Agent 场景里必须被容忍和管理

- 02:09:03 另类架构与多模态：统一不是目的，智能和效率才是目的
  - 音频路线偏向离散 token 建模
    - 团队希望把音频变成类似文本的离散 token，并尽量做到无损重建
    - 这需要多层 RVQ、更长预训练和更高研究成本
    - 离散化带来的泛化能力，在 TTS 的自然语言 style 标签遵循上有所体现
  - 图片和视频仍在权衡
    - Omni 的视觉侧仍是连续表征，只是把 VIT 做成更高效的 Hybrid sliding window 结构
    - 她不愿为统一架构牺牲模型智能水平
    - Agent 辅助重写 infra 后，为统一而统一的必要性下降
  - 多模态对智能的作用仍需保留疑问
    - 原生多模态让 Omni 在世界知识、细节感知和情绪理解上有体感优势
    - 现有 benchmark 很难捕捉这种感知力
    - 她更确定的是 Agent 行动需要多模态，较谨慎的是多模态是否必然促进 AGI

- 02:22:32 AGI 之路：模型进化环境不同于人类进化
  - AI 没有生存危机
    - 人类智能来自自然环境和生存压力，模型一开始拿到的是算力、人类知识和大量协助
    - 没有生存压力让模型进化更自由，也更不受人类路径约束
    - 人类智能三角形不能简单套到大模型演化上
  - Coding 之后是更广泛生产力场景
    - Coding 会继续沿着更复杂软件工程任务推进
    - 机器人会成为 Agent 跳出屏幕的交互方式，但硬件、电池和灵巧手会拖慢现实世界速度
    - 工作模式会先被颠覆，生活模式需要更多硬件成熟后才会被深度改变
  - 时间线被明显提前
    - 她两个月前认为 AGI 至少两年以上，现在判断两年内有可能
    - 关键变量是 AI 能否参与训练更强 AI，并产生新的研究
    - 国内具备 1T 基座的团队，未来两三个月会进入加速追赶窗口

  - AGI 后人仍要参与科学研究

- 02:48:34 过去三年的模型进化史：从 Chat 到 Reasoning 再到 Agent
  - 2022 是短上下文 Chat 的激发
    - ChatGPT 用对话交互让人感受到 4K 左右 context 里的智能
    - 交互形态让模型能力被大众看见
    - 当时的智能主要发生在短上下文和多轮对话纠错中
  - 2023 是开源追赶预训练
    - Llama 打开训练范式，Qwen 推动全尺寸和生态，DeepSeek 更重视结构创新和有限算力下的 scaling
    - Qwen 的生态价值和 DeepSeek 的研究价值互相促进
    - MoE、MLA 等结构让训练成本和推理成本重新被芯片行业理解
  - 2024 的 R1 是后训练范式转折
    - R1 的核心启发不是单个结果，而是团队如何为后训练重组
    - Code 和 Math 的可验证任务证明 RL 可以从垂直场景泛化
    - 她从中学到，垂直场景里的范式变化可能被低估
  - 2025 是 Chat Reasoning 和 Agent 的分叉
    - 一条路继续在 chat reasoning benchmark 上深耕
    - 另一条路转向复杂 Agent 框架、端到端任务和真实可用性
    - 她认为只在特定 benchmark 上提升，不代表模型真的能在真实 Agent 场景里可用

- 03:05:54 当下共识与竞争：Anthropic 路径被重新理解
  - 国内团队进入加速追赶
    - Pre-train 代差已经很小，甚至国内在某些结构效率上有优势
    - 真正差异在 Agent post-train 和 RL scaling
    - Anthropic 过去的 context engineering 和 Agent 框架设计，可能本来就在配合模型完成复杂任务
  - 竞争维度和速度都增加
    - 预训练很难一个月出一个模型，后训练可以更快迭代
    - Agent 竞争还要看推理侧结构、芯片供给、长上下文策略和框架演进预判
    - 推理需求可能随着框架增强和模型成本下降出现数倍到十倍增长
  - 模型公司边界变模糊
    - 模型借助 Agent 框架本身又变成新的产品
    - DAU 叙事不能直接服务于让模型完成更高价值任务
    - token usage、高价值任务完成率和资源调度能力，更接近她理解的竞争目标

- 03:12:37 创业、开源和胜负手：生态位决定开放策略
  - 创业公司团队规模可能变小
    - Agent 会让几个人甚至一个人拥有更接近公司的生产力
    - 关键前提是模型成本足够低，multi-agent 架构足够成熟
    - 当前 multi-agent 还更像节省成本和时间的工具，离替代员工仍差每个环节的一点点
  - 开源是加速 Agent 进程
    - 如果 Agent 会替代大部分生产力，算力、芯片和模型部署都会高度分散
    - 开源有助于框架、芯片、能源和模型生态共同推进
    - 公司是否敢开源，取决于有没有模型之外的战略生态位
  - 2026 胜负手
    - 基座预训练不能错，尤其是 1T 以上和 coding 潜能
    - Agent 框架和模型要互相自迭代
    - Agent 架构要能理解和调度公司已有资源，例如 OS、硬件、流量、社交和生态
    - 公司需要重新思考原有人员和流程是否仍然必要

- 03:19:45 环境比经验更重要：Frontier Lab 的组织设计
  - Frontier 来自工业级原创
    - 她看重能 scaling 到工业级模型的原创研究，而不是只产出 paper
    - DeepSeek-V3 和 MiMo-V2 都属于在资源约束和范式预判中做出的工业级模型
    - 信息来源更多来自自己做实验和团队内部交流，而不是追学术会议
  - 经验可以被环境快速刷新
    - 大模型训练经验最多一两个月、慢则三四个月可以被新环境洗出来
    - 她更看重基础能力、好奇心、热爱和上限，而不是已经被 supervision 到哪个状态
