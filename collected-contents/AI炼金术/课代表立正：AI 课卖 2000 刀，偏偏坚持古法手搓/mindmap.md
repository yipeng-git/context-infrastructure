# 课代表立正：AI 课卖 2000 刀，偏偏坚持古法手搓

- 00:00 AI 时代的稀缺价值：先有可放大的东西
  - AI 是放大器，不是起爆器
    - 只会 AI 很难形成独立价值，AI 需要放大已有业务、经验或判断
    - 没有 AI，Superlinear Academy 收入可能少 20%，不是从零到一的差别
    - 人脑里的隐性知识需要主动沉淀成 context 和 conviction
  - 正确的非共识是人的核心贡献
    - 共识没有高价值，错误的非共识也没有价值，关键是 correct contrarian
    - AI 训练于多数人的数据，很难让用户相信真正的非共识
    - 人的价值在于选定少数路径，持续 build 到别人开始相信
  - 打工人要先修正商业认知
    - 从 0 到 1 之前，先从 -10 到 0，修正对价值和市场的错误理解
    - 更难的东西更有价值，是做题家进入市场后的高风险假设

- 01:21 从创业者到服务公司：产品形态服从价值交付
  - 创业者不一定要做 App 或 SaaS
    - 立正把公司定义为围绕价值提供服务，课程、社区和工具只是不同形态
    - 企业培训、课程和 YouTube 会员共同构成收入，而非单一产品线
    - 服务的优势在于交付确定性，能承诺第二天提升生产力
  - 买方视角和供给方视角不同
    - 买方容易相信工具更确定，供给方更清楚工具未必被正确使用
    - Agency 或培训能把工具使用、需求澄清和结果交付包起来
    - 高客单价来自对结果的确定性，而不是工具功能本身

- 05:43 2000 美元 AI 课的反直觉定价逻辑
  - 高价值环节刻意不用 AI
    - 课程由人手写，问答由人回答，因为用户为用心、感知和品牌付费
    - 自动化内容一旦被标注为 AI 生成，C 端愿付价格会快速下降
    - 企业培训可以大量使用 AI，因为客户购买的是定制结果
  - AI 用在不损害品牌溢价的后台
    - Deep news、翻译和调研总结可以 AI 化，因为它们是知识供给基础设施
    - 核心课程和答疑保持人工，是为了保留高客单价服务感
    - 这不是反 AI，而是区分效率层和溢价层
  - 商业模式由内容、课程、社区和服务组成
    - 团队核心是立正和亚哥，配少量增长和社区运营兼职
    - 约 90% 收入来自课程，YouTube 广告和会员只是补充
    - 2B 今年投入增加，精力可能 2B:2C 为 8:2，收入仍可能 2:8
  - 美国知识付费环境帮助建立高定价
    - 先在 Maven 等高客单价平台站住脚，再回到中文市场
    - Google、Meta、OpenAI、Anthropic 学员和 4.9/5 review 提供信任资产
    - 公开社区里的学员成果成为获客证据，而不是靠空喊课程很强

- 14:58 从 user 到 architect：AI 学习的四个台阶
  - 三个实践门槛
    - 第一关是不进健身房：看很多 AI 内容，但不每天打开 Cursor 或 Claude Code
    - 第二关是从单点使用到管线：能描述需求、控制质量、让 AI 自己跑完任务
    - 第三关是从小工具到 agentic 系统：能构建有复杂度、可上线、可运维的产品
  - 四个成长阶段
    - User 会使用 ChatGPT 或现成软件，但主要停留在问答和消费
    - Tinker 会用 Cursor / Claude Code 做小东西，但缺少质量控制和工程方法
    - Builder 会用 evaluation、context、documentation 和 pipeline 可靠完成工作
    - Architect / Conductor 能编排模型、agent 和工具，解决复杂系统问题
  - 学 AI 的顺序不是工具优先
    - 工具人人可得，方法需要底层理解，习惯决定是否持续使用
    - 课程先处理 mindset，因为没有判断原点就拿不住方法
    - AI 用不好时，很多问题来自需求表达、反馈循环和上下文管理，而不是编程水平
  - 程序员要从过程确定性转向结果确定性
    - 过程确定性关心输入、AI 调用和输出链路是否按步骤执行
    - 结果确定性关心目标输出，并构造 feedback loop 迭代到达标准
    - 未来的角色更像管理者和领航者，而不是继续追求划桨速度

- 25:04 正确的非共识与技术判断：不要追逐模糊概念
  - 人与 AI 的差异会落到 context
    - 同一个 AI 系统拿到不同人的 context，输出的文章、产品和设计会不同
    - 隐性知识需要被提炼成 AI 可用的 context architecture
    - taste、agency 和 conviction 最后都要进入系统上下文才产生差异
  - 课程 12 次迭代后 80% 不变
    - 不教工具 tutorial，而讲 context window、evaluation、documentation 和软件工程原理
    - RAG、Framework、MCP 等热点都要判断该不该作为学习入口
    - 对技术本质的理解比追热点更能降低课程维护压力
  - 模糊概念要被拆成可操作定义
    - Prompt / context / agent 相关概念如果指向太宽，会降低讨论效率
    - Agent 至少包含多步决策、自我迭代和工具调用三个特征
    - Compound 是结果，不是具体方法；要落到 agentic core、低摩擦交互、contextual 和 proactive
  - Manus 的优势和短板
    - 优势是 IM 交互、统一上下文、记忆系统、生态和主动性，降低普通人门槛
    - 短板是黑盒、不可观测、记忆不可修改、上下文不可隔离
    - 更可控的替代是文件夹 + Claude Code / Cursor + context architecture + 可编辑 memory

- 41:30 个人提效和组织提效不是一回事
  - 好项目从真实需求开始
    - 社区要求学员从真实需求出发，作品通常有身边的真实用户
    - Share projects 区域提供一万多人规模的 go to market 测试场
    - 做出来只是第一步，找用户、验证需求和增长同样需要训练
  - 组织提效受三层约束
    - 流程没有围绕 AI 重设，只替换单个环节，整体效率提升有限
    - 团队 context 不共享，每个人的 AI、prompt、工作流和认知彼此割裂
    - 组织结构为专家分工而设计，但 AI 更适合 end-to-end 小 pod
  - 大公司最难处理的是生产关系
    - 小公司提效三五倍常见，大公司可能看不到实质成本下降
    - 人多事少时，AI 提效会让人更多事更少，缺少学习压力
    - 砍掉一半人能制造 shock、进入人少事多状态、打破岗位分工，但代价极高
  - 更稳妥的路径是开新业务
    - 老业务运行正常时，既有团队能列出 AI 的一百个缺点
    - 新业务和人手不足会迫使团队寻找 AI 能用的部分
    - 孵化器式小团队更容易形成 end-to-end owner

- 49:14 社区学员案例：AI 创业不必等于 VC 项目
  - 垂直数据库和工具有现实商业价值
    - Obsidian 插件、考试资料聚合站、法律查询工具都能形成小规模收入
    - 很多需求过去只能开淘宝店卖资料，现在可以做成带 AI 查询的网站
    - Web coding 的价值在于把旧需求快速产品化，而不是追求炫技
  - AI 也能参与增长
    - 网站嵌入 analytics 后，让 AI 分析 SEO、增长和变现建议
    - 亚哥博客通过类似方法一个月浏览量增长约 5 倍
    - 从个人工具到商业项目，中间缺的往往是 go to market 训练
  - 重来一遍会更早商业化
    - 目标不是单纯做内容，而是建立个人品牌，让别人知道你能解决什么问题
    - 商业化不一定接商单，可以是个人品牌 + 满足具体需求
    - Service、产品或其他形态都应根据市场反馈调整

- 52:55 给打工人的转型建议：从市场价值倒推能力
  - 先从 -10 到 0
    - 大厂和学校训练出的是解题能力，但市场先问这道题值多少钱
    - 真实世界的题目难度和报酬不成正比，选题比答题更重要
    - 要学习市场语言，而不是继续用考试分数理解世界
  - 从价值出发，不从专业出发
    - 问能不能帮奶茶店销售额增长 30%，比问数据科学能做什么更接近市场
    - 很多工作技能只在公司内部有价值，换到外部客户未必是通货
    - 能变现的能力，是隔壁老王也愿意付钱购买的能力
  - 安全垫降低错误决策概率
    - 大厂打工阶段应多存钱，给自己一两年探索空间
    - 绝望状态下更容易被割韭菜，也更容易做短视决定
    - 经营者视角也能反过来帮助在公司里选择更有价值的工作
  - 沉没成本不是成本
    - 立正判断数据科学岗位会被 AI 取代后，转去 Statsig 做销售和 evangelist
    - 挂着数据科学家 title 是为了卖 A/B 实验产品更有可信度
    - 从数据科学家变成餐馆老板也可以成立，关键是面向未来价值重新选择

- 01:03:44 给 AI 炼金术的商业化建议
  - 先建立确定性触达关系
    - 播客比短视频和长视频更强，但仍依赖平台分发
    - 微信群、社区、邮件列表或私域能让观众确定性找到你
    - 没有私域关系，内容资产会被平台规则拿捏
  - 2B 广告可能比卖课更适合
    - 如果主理人对钱的权重低于生活方式，卖课未必是最佳选择
    - 面向创业者生态的招聘、品牌和影响力广告更顺手
    - 可以通过 newsletter 或广告位小试，追踪转化数据
  - 2B 培训要产品化成 solution
    - 按时间卖咨询难以规模化，产品化培训、平台、软件或服务团队更有企业价值
    - 企业愿意买单需要一个清晰理由，而不是只听一次道理
    - 高价服务需要绝活感：惊艳、稀缺、别处买不到
  - 2000 美元课的绝活组合
    - 高端平台背书、顶级公司学员、4.9/5 review 和低退款率形成信任基础
    - 内容、社区、学员成果、Builder Space 和免费深度文章共同提高价值感
    - 高价格本身提高 commitment，用户付得越多，越可能认真投入并得到效果
